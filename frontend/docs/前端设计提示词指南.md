# 前端设计提示词指南

## 文档信息
- 文档名称：前端设计提示词指南
- 版本号：v1.0.0
- 创建日期：2025-12-09
- 用途：为创建炫酷前端页面提供提示词建议和设计指导

## 1. 视觉设计类提示词

### 1.1 整体风格

#### 玻璃态（Glassmorphism）风格
```
创建一个现代、极简、充满活力的兴趣追踪仪表板，采用玻璃态（Glassmorphism）设计风格，带有柔和的毛玻璃效果和微妙的阴影
```

#### 深色科技风
```
使用深色主题配合霓虹色强调，营造科技感和未来感，适合展示摄影、影视、音乐等多媒体内容
```

#### 渐变动态风
```
采用渐变背景和动态粒子效果，营造沉浸式体验，同时保持内容清晰可读
```

#### 极简现代风
```
采用极简主义设计，大量留白，使用细线条和柔和的阴影，突出内容本身
```

### 1.2 色彩方案

#### 深色渐变方案
```
使用深色背景（#0a0e27）配合渐变色（紫色到蓝色渐变 #667eea → #764ba2），强调色使用明亮的青色（#00d4ff）
```

#### 柔和粉紫方案
```
采用柔和的粉紫色调（#f5f7fa → #c3cfe2），配合温暖的橙色（#ff6b6b）作为行动按钮
```

#### 深蓝金色方案
```
使用深色模式，主色调为深蓝（#1a1a2e），配合金色（#f4d03f）作为高亮色
```

#### 活力渐变方案
```
使用从深紫（#667eea）到粉红（#f093fb）的渐变，配合白色文字，营造年轻活力的氛围
```

## 2. 交互与动画类提示词

### 2.1 页面过渡动画

#### 流畅切换
```
实现流畅的页面切换动画，使用淡入淡出和轻微缩放效果，配合视差滚动
```

#### 3D翻转效果
```
添加卡片悬停时的3D翻转效果和阴影变化，增强交互反馈
```

#### 骨架屏加载
```
使用骨架屏（Skeleton Screen）加载动画，提升加载体验，减少用户等待焦虑
```

#### 页面转场
```
使用滑动转场效果，新页面从右侧滑入，旧页面向左滑出，配合淡入淡出
```

### 2.2 数据可视化动画

#### 动态统计卡片
```
创建动态的统计卡片，数字滚动动画，圆形进度条展示占比，使用Chart.js或D3.js实现流畅的图表动画
```

#### 时间轴动画
```
时间线视图采用垂直时间轴设计，带有连接线和节点动画，支持无限滚动加载，节点带有脉冲动画效果
```

#### 照片墙动画
```
照片墙采用瀑布流布局，带有懒加载和淡入动画效果，悬停时轻微放大并显示详细信息
```

#### 图表动画
```
使用ECharts或Recharts创建交互式图表，带有数据加载动画和悬停提示，支持缩放和平滑过渡
```

## 3. 组件级提示词

### 3.1 输入组件

#### 智能搜索框
```
设计一个智能搜索框，带有自动补全、搜索历史，以及搜索时的脉冲动画效果，支持键盘快捷键
```

#### 标签选择器
```
创建一个可拖拽的标签选择器，支持多选，带有颜色标签和删除动画，支持自定义标签颜色
```

#### 文件上传组件
```
设计一个照片上传组件，支持拖拽上传，带有上传进度条和预览功能，支持多文件上传和批量处理
```

#### 评分组件
```
创建一个交互式评分组件，使用星星或数字滑块，带有悬停效果和动画反馈
```

### 3.2 展示组件

#### 卡片组件
```
设计一个通用卡片组件，带有悬停阴影效果、点击波纹效果，支持不同尺寸和布局
```

#### 模态框
```
创建一个优雅的模态框组件，带有背景模糊效果、从中心弹出的动画，支持ESC关闭和点击外部关闭
```

#### 通知提示
```
设计一个通知系统，支持成功、错误、警告、信息四种类型，带有从顶部滑入的动画和自动消失功能
```

#### 加载状态
```
创建多种加载状态：骨架屏、旋转加载器、进度条、脉冲动画，根据场景选择合适的加载方式
```

## 4. 状态反馈提示词

### 4.1 AI处理状态

#### AI标签生成
```
AI标签生成时显示优雅的加载动画，使用粒子效果或旋转的图标，配合文字提示"AI正在分析中..."
```

#### AI处理成功
```
AI处理成功时显示绿色对勾动画，标签逐个淡入显示，带有轻微的弹跳效果
```

#### AI处理失败
```
AI处理失败时显示友好的错误提示，提供重试按钮和手动输入选项，使用柔和的红色提示
```

### 4.2 操作反馈

#### 成功反馈
```
操作成功时显示Toast通知，带有绿色对勾图标和淡入淡出动画，3秒后自动消失
```

#### 错误反馈
```
错误状态使用柔和的红色提示，带有轻微震动动画，提供明确的错误信息和解决方案
```

#### 空状态
```
空状态页面设计精美的插画，配合引导性文案和行动按钮，使用柔和的灰色调
```

## 5. 技术实现类提示词

### 5.1 技术栈选择

#### React生态
```
使用React + TypeScript + Tailwind CSS构建，配合Framer Motion实现流畅动画，使用React Query进行数据管理
```

#### Next.js方案
```
使用Next.js实现SSR，配合React Query进行数据管理，使用Zustand进行状态管理，使用shadcn/ui组件库
```

#### 3D效果
```
使用Three.js或React Three Fiber添加3D背景效果，使用GSAP实现复杂动画序列
```

#### Vue生态
```
使用Vue 3 + TypeScript + Vite构建，配合Pinia进行状态管理，使用VueUse工具库，使用Element Plus或Naive UI组件库
```

### 5.2 性能优化

#### 虚拟滚动
```
实现虚拟滚动优化长列表性能，使用react-window或react-virtualized，支持动态高度
```

#### 懒加载
```
使用Intersection Observer实现图片懒加载，配合模糊占位图（Blur Placeholder）提升体验
```

#### 主题切换
```
使用CSS变量实现主题切换，支持深色/浅色模式平滑过渡，保存用户偏好到localStorage
```

#### Web Workers
```
使用Web Workers处理大量数据计算，保持UI流畅响应，避免阻塞主线程
```

## 6. 针对项目的具体场景提示词

### 6.1 首页仪表板

```
创建一个响应式仪表板，顶部是导航栏，主体分为三列：
- 左侧是快速操作卡片（上传照片、搜索影视等），带有图标和悬停效果
- 中间是时间线预览，显示最近的活动记录
- 右侧是统计卡片（今日记录数、本周趋势、类型占比饼图）

使用卡片式布局，带有悬停效果和微妙的阴影。支持拖拽调整布局。
```

### 6.2 照片展示页

```
设计一个照片网格布局，支持瀑布流（Masonry Layout），每张照片带有：
- EXIF信息浮层（悬停显示设备、时间、GPS等信息）
- 标签以彩色徽章形式展示在照片底部
- 点击进入全屏查看模式

顶部有过滤器和排序选项，使用粘性定位。支持全屏查看模式，带有左右切换动画和缩放功能。
```

### 6.3 影视/音乐列表页

```
创建一个卡片式列表布局，每个卡片包含：
- 封面图片（带有加载动画）
- 标题、副标题（导演/艺术家）
- 状态标签（想看/在看/已看/弃剧）
- 评分显示（星级或数字）
- 标签列表

支持网格视图和列表视图切换，带有平滑的过渡动画。
```

### 6.4 时间线页面

```
创建一个垂直时间线，设计特点：
- 左侧显示日期，使用大号字体突出
- 右侧显示不同类型的记录卡片（照片、影视、音乐、球赛）
- 每种类型有独特的图标和颜色标识
- 使用连接线连接同一天的记录

支持按时间、类型、标签过滤，带有平滑的滚动动画和加载更多功能。支持时间轴缩放（日/周/月视图）。
```

### 6.5 搜索页面

```
设计一个搜索页面，包含：
- 大型搜索框，带有搜索图标和清除按钮
- 搜索历史记录（最近5条）
- 实时搜索建议（防抖处理）
- 搜索结果分类展示（照片/影视/音乐/球赛）
- 空状态提示和加载状态

搜索结果带有高亮显示，支持键盘导航。
```

### 6.6 统计页面

```
创建一个数据可视化页面，包含：
- 顶部统计卡片（总数、本周新增、本月新增）
- 类型分布饼图（可点击查看详情）
- 时间趋势折线图（支持时间范围选择）
- 标签云图（热门标签）
- 活动热力图（按日期显示活动强度）

所有图表支持交互，带有动画效果和详细提示。
```

## 8. 最佳实践建议

### 8.1 性能优化
- 使用代码分割（Code Splitting）减少初始加载时间
- 图片使用WebP格式，配合懒加载
- 使用CDN加速静态资源
- 合理使用缓存策略
- 避免不必要的重渲染

### 8.2 可访问性
- 确保键盘导航可用
- 提供足够的颜色对比度（WCAG AA标准）
- 为图片添加alt文本
- 使用语义化HTML标签
- 支持屏幕阅读器

### 8.3 响应式设计
- 移动优先（Mobile First）设计
- 使用Flexbox和Grid布局
- 设置合理的断点（320px, 768px, 1024px, 1440px）
- 测试不同设备和浏览器
- 考虑横屏和竖屏布局

### 8.4 用户体验
- 提供即时反馈（按钮点击、表单提交等）
- 使用加载状态减少用户焦虑
- 错误提示清晰明确
- 支持撤销操作
- 保持操作流程简洁

## 9. 提示词使用技巧

### 9.1 组合使用
将多个提示词组合使用，例如：
```
创建一个采用玻璃态设计的响应式仪表板，使用深色渐变背景（#0a0e27 → #667eea），
配合Framer Motion实现流畅动画，包含动态统计卡片和时间线视图
```

### 9.2 具体化描述
越具体的描述越好，例如：
```
创建一个照片网格布局，使用CSS Grid实现，每行3列（桌面端），
卡片带有圆角（8px）和阴影（0 4px 6px rgba(0,0,0,0.1)），
悬停时放大1.05倍并显示EXIF信息浮层
```

## 10. 示例完整提示词

### 示例1：首页仪表板
```
使用Next.js + React + TypeScript + Tailwind CSS创建一个兴趣追踪应用的首页仪表板。

设计要求：
- 采用深色主题，背景使用深蓝到紫色的渐变（#1a1a2e → #16213e → #0f3460）
- 顶部导航栏使用玻璃态效果，带有模糊背景和半透明
- 主体分为三列布局（响应式，移动端单列）
  - 左侧：快速操作卡片（上传照片、搜索影视、搜索音乐、添加球赛），每个卡片带有图标和悬停效果
  - 中间：时间线预览，显示最近5条记录，使用垂直时间轴设计
  - 右侧：统计卡片区域，包含今日记录数、本周趋势图、类型占比饼图

交互要求：
- 所有卡片带有悬停阴影效果和轻微缩放
- 使用Framer Motion实现页面加载动画和卡片进入动画
- 统计数字使用滚动动画（count-up effect）
- 支持深色/浅色主题切换，使用CSS变量实现平滑过渡

技术栈：
- Next.js 14 (App Router)
- React 18
- TypeScript
- Tailwind CSS
- Framer Motion
- Recharts (图表)
- shadcn/ui (组件库)
```

### 示例2：照片展示页
```
创建一个照片展示页面，使用React + TypeScript + Tailwind CSS。

布局设计：
- 顶部：粘性导航栏，包含标题、过滤器（类型、标签、时间范围）、排序选项、视图切换（网格/列表）
- 主体：瀑布流布局（Masonry Layout），使用react-masonry-css实现
- 每张照片卡片：
  - 图片使用懒加载，带有模糊占位图
  - 底部显示标签（彩色徽章）
  - 悬停时显示EXIF信息浮层（设备、拍摄时间、GPS位置）
  - 点击进入全屏查看模式

交互效果：
- 图片加载时使用淡入动画
- 悬停时卡片轻微放大（scale 1.02）并显示阴影
- 全屏模式支持左右切换、缩放、旋转
- 使用Intersection Observer实现无限滚动

视觉风格：
- 使用卡片式设计，圆角8px，白色背景（深色模式下深灰）
- 标签使用渐变色，不同标签不同颜色
- 支持深色模式
```
