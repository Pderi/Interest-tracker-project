# AI助手提示词模板 - 前端开发专用

## 文档说明
本文档提供了与AI助手协作开发前端项目时的最佳提示词模板，帮助AI更好地理解需求并生成高质量代码。

---

## 一、项目初始化类提示词

### 1.1 完整项目初始化（推荐）

```
请帮我初始化一个Vue 3前端项目，项目信息如下：

【项目基本信息】
- 项目名称：兴趣追踪平台 (Interest Tracker)
- 技术栈：Vue 3 + TypeScript + Vite
- UI框架：Element Plus 或 shadcn/ui（请选择最适合的）
- 状态管理：Pinia
- 路由：Vue Router
- HTTP客户端：Axios
- 构建工具：Vite
- 代码规范：ESLint + Prettier

【项目结构要求】
- 使用标准的Vue 3项目结构
- 目录组织：src/views（页面）、src/components（组件）、src/api（接口）、src/store（状态管理）、src/utils（工具函数）
- 支持TypeScript类型定义
- 配置路径别名（@指向src目录）

【功能模块】
1. 用户认证（登录/注册）
2. 摄影模块（照片上传、列表、详情）
3. 影视模块（TMDB搜索、记录管理）
4. 音乐模块（Last.fm搜索、记录管理）
5. 球赛模块（比赛记录）
6. 时间线/统计页面
7. 个人中心

【设计要求】
- 响应式设计（支持桌面端和移动端）
- 深色/浅色主题切换
- 使用Tailwind CSS或Element Plus的样式系统
- 支持国际化（i18n，中英文）

【后端接口】
- 基础URL：http://localhost:8080/api
- 统一返回格式：CommonResult<T>
- 认证方式：JWT Token（存储在localStorage）

请创建完整的项目结构，包括：
1. package.json和依赖配置
2. vite.config.ts配置文件
3. tsconfig.json TypeScript配置
4. 基础路由配置
5. Axios拦截器（请求/响应处理、Token管理）
6. Pinia store基础结构
7. 环境变量配置（.env.development, .env.production）
8. 基础布局组件（Header、Sidebar、Main）
9. 登录页面基础结构
10. README.md项目说明文档
```

### 1.2 快速初始化（简化版）

## 三、功能实现类提示词

### 3.1 API接口封装

```
请帮我封装【模块名称】相关的API接口，后端接口规范如下：

【接口列表】
1. 获取列表
   - 方法：GET
   - 路径：/api/xxx/list
   - 参数：{ page: number, size: number, keyword?: string }
   - 返回：CommonResult<PageResult<XxxVO>>

2. 创建记录
   - 方法：POST
   - 路径：/api/xxx/create
   - 参数：XxxCreateReq
   - 返回：CommonResult<Long>

3. 更新记录
   - 方法：PUT
   - 路径：/api/xxx/update
   - 参数：XxxUpdateReq
   - 返回：CommonResult<Boolean>

4. 删除记录
   - 方法：DELETE
   - 路径：/api/xxx/delete
   - 参数：{ id: number }

【技术要求】
- 使用TypeScript定义请求和响应类型

- 统一错误处理
- 支持请求取消

请提供：
```

```
请帮我创建一个照片上传页面，功能要求：
```

## 九、注意事项

1. **提供完整信息**：尽量提供项目背景、技术栈、需求细节
2. **分步骤进行**：复杂功能分步骤实现，不要一次性要求太多
3. **及时反馈**：如果生成的代码不符合预期，及时说明问题
4. **保持一致性**：使用统一的代码风格和项目规范
5. **测试验证**：生成代码后及时测试，发现问题及时反馈
