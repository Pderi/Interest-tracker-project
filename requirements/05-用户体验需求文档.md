# 用户体验需求文档（UXD）

## 1. 文档信息
- 文档名称：用户体验需求文档（UXD）
- 版本号：v1.0.0
- 创建日期：2025-12-09
- 最后更新：2025-12-09
- 作者：设计/产品团队
- 状态：草稿

## 2. 文档目的
- 以用户体验为中心，定义信息架构、核心流程、交互与可用性要求，为设计与实现提供依据。
- 对齐业务目标：提升记录效率、降低手动成本、提高回顾与留存。

## 3. 用户与场景
- 角色分层：
  - 轻度用户：零散记录，期望“即用即走”、自动化和简单回顾。
  - 中度用户：关注标签/检索/过滤，期望较好的整理与浏览体验。
  - 重度用户：批量管理、数据准确性、导出/统计（本期先满足基础）。
- 核心场景：
  - 上传/创建记录（照片/影视/音乐/球赛简版）
  - 自动标签与手动补充
  - 列表/详情查看，过滤与排序
  - 时间线/基础统计卡片回顾
  - 登录/登出与权限提示

## 4. 设计原则
- 低摩擦：少步骤、少必填、清晰提示，优先自动化。
- 一致性：组件/文案/校验/状态风格一致。
- 可感知：加载、错误、AI 处理中均有可见反馈；失败有兜底。
- 可编辑：AI 结果可编辑/覆盖，用户可控。
- 可访问：基础响应式，键盘可操作；颜色/对比度符合常规可读性（WCAG 基本要求）。

## 5. 信息架构（IA）
```
首页/概览（可选基础） 
├── 记录入口：摄影 / 影视 / 音乐 / 球赛
├── 快捷入口：上传照片、搜索影视/音乐、添加比赛
├── 基础统计卡片：今日/本周记录数、类型占比（简版）
├── 时间线入口

摄影
├── 列表（支持过滤/排序/搜索）
├── 上传（含自动标签开关）
├── 详情（EXIF、标签、描述、时间/地点）

影视
├── 列表（过滤/排序/搜索）
├── 搜索 TMDB → 选择 → 创建
├── 详情（状态/评分/标签/描述）

音乐
├── 列表（过滤/排序/搜索）
├── 搜索 Last.fm → 选择 → 创建
├── 详情（歌名/艺术家/专辑/标签）

球赛（简版）
├── 列表（过滤/排序）
├── 手动添加比赛
├── 详情（对阵/比分/时间/类型）

时间线/回顾
├── 时间线视图（按时间聚合）
├── 列表视图（时间排序）
├── 基础统计卡片
```

## 6. 核心流程（摘要）
1) 上传照片 → 自动标签开关默认开启 → 显示“AI 处理中” → 标签可编辑 → 保存  
2) 搜索影视（TMDB）→ 选择结果 → 自动填充基础信息 → 可编辑 → 保存  
3) 搜索音乐（Last.fm）→ 选择结果 → 自动填充 → 可编辑 → 保存  
4) 查看列表/时间线 → 过滤（类型/标签/时间）→ 排序（时间/评分）  
5) 基础统计卡片 → 查看数量/时间分布/占比（简版）  
6) 登录/登出 → 未登录访问受限页面时提示登录  

## 7. 交互与状态要求
- 加载：列表加载、AI 处理、搜索结果均需 Loading 状态。
- 空态：无数据/无搜索结果/AI 失败需提供指引或重试入口。
- 错误：接口/AI 失败显示可理解文案；提供重试/手动输入。
- 反馈：创建/保存成功提示；危险操作需确认。
- 可编辑：AI 标签、评分、状态、描述均可编辑；编辑后支持保存/取消。
- 过滤/排序：过滤与排序组合使用时结果正确；提供清空筛选。

## 8. 表单与校验
- 必填：标题（或默认生成）、来源选择（搜索/手动）、时间字段（记录时间/比赛时间）。
- 校验：文件类型/大小、评分范围、状态合法性、时间格式。
- 文案：错误与提示用语简洁明确，避免模糊术语。

## 9. 可用性要求
- 步骤最少化：上传/创建不超过必要步骤；默认值与自动填充优先。
- 搜索体验：输入时可提示；失败时允许手动录入。
- 响应式：基础适配桌面/常见笔电分辨率；移动端基础可用（MVP 不深度适配）。
- 可访问性：键盘可聚焦与操作核心表单；按钮/链接有可见焦点；颜色对比度满足基本可读性。

## 10. 内容与文案
- 统一用词：记录、标签、评分、状态（想看/在看/已看/弃剧），上传/保存/取消/重试。
- AI 状态文案：处理中/失败可重试/可手动编辑。
- 空态文案：明确下一步操作（如“去上传/去搜索/重试”）。

## 11. 视觉与组件（概要）
- 组件：表单（输入、选择、日期、评分）、列表卡片、标签、多选、上传组件、Loading/Empty/Error 状态组件。
- 视觉基调：简洁、清晰，突出内容本身；状态/反馈颜色一致。
- 图标与缩略图：照片/影视/音乐使用封面或占位图；球赛使用简单对阵展示。

## 12. 埋点与体验指标（占位）
- 关键埋点：上传/创建成功、搜索成功/失败、AI 调用成功/失败、过滤/排序使用、时间线/统计查看、登录/登出。
- 体验指标（后续与数据确认）：表单放弃率、AI 成功率、搜索成功率、回顾页面停留时长、7 日/30 日留存贡献。

## 13. 约束与假设
- 免费 API 优先，可能存在响应时延；需用 Loading 和重试兜底。
- 付费 AI 默认关闭；不保证高并发下的 AI 体验。
- MVP 优先桌面 Web，移动端仅基础可用。

## 14. 验收标准（体验向）
- 必填/校验清晰，错误提示可理解；关键流程可完成且无阻塞。
- 过滤/排序/搜索结果正确；空态与失败有指引/重试。
- AI 标签可编辑，失败可兜底；上传/创建成功有反馈。
- 关键埋点生效（后续对齐数据方案）。

## 15. 追溯与变更
- 与 BRD/PRD/FR 映射：核心流程、必填校验、状态反馈需与功能需求一致。
- 变更：记录原因、影响范围；评审后更新版本号与历史。
