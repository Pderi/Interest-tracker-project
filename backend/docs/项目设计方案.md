# 个人兴趣内容管理与分析平台 - 详细设计方案

## 项目概述

**项目名称：** 兴趣足迹 (Interest Tracker) / 生活记录 (Life Logger)

**核心理念：** 一个整合摄影、影视、音乐、球赛四大兴趣领域的个人内容管理与数据分析平台，通过数据可视化帮助用户了解自己的兴趣习惯，发现更多相关内容。

---

## 一、核心功能模块详细设计

### 1.1 摄影作品管理模块 📸

#### 基础功能
- **照片上传**
  - 支持单张/批量上传
  - 图片压缩与缩略图生成
  - 支持常见格式（JPG、PNG、RAW等）
  
- **照片信息管理**
  - EXIF数据自动提取（拍摄时间、设备、ISO、光圈、快门等）
  - 手动补充：标题、描述、标签、拍摄地点
  - 照片分类：按主题、地点、设备、时间等
  
- **照片展示**
  - 瀑布流布局
  - 时间线视图
  - 地图视图（基于GPS信息）
  - 相册集合功能

#### 创新功能
- **拍摄数据分析**
  - 拍摄频率统计（按日/周/月/年）
  - 常用设备/镜头统计
  - 拍摄参数偏好分析（ISO、光圈范围）
  - 拍摄地点热力图
  - 最佳拍摄时间段分析
  
- **照片质量评估**
  - 基于EXIF数据的照片质量评分
  - 构图建议（基于主流摄影规则）
  - 相似照片对比功能

---

### 1.2 影视剧追踪模块 🎬

#### 基础功能
- **观看记录管理**
  - 添加观看记录（电影/剧集）
  - 标记观看状态：想看、在看、已看、弃剧
  - 评分系统（1-10分，可精确到0.5分）
  - 观看日期记录
  
- **影视信息管理**
  - 影片信息自动获取（可接入第三方API）
  - 自定义标签分类
  - 个人评价/短评
  - 观看进度追踪（针对剧集）

#### 创新功能
- **观影数据分析**
  - 观影时长统计（日/周/月/年）
  - 类型偏好分析（动作、科幻、爱情等）
  - 评分分布统计
  - 观影时间分布（工作日vs周末）
  - 观影日历视图
  
- **智能推荐**
  - 基于已看影片推荐相似内容
  - 基于评分模式推荐
  - 热门未看影片提醒
  
- **观影成就系统**
  - 连续观影天数
  - 年度观影数量
  - 类型探索奖章

---

### 1.3 音乐管理模块 🎵

#### 基础功能
- **音乐收藏管理**
  - 歌单创建与编辑
  - 歌曲收藏（本地文件或在线链接）
  - 歌手/专辑关注
  
- **播放记录**
  - 播放历史记录
  - 播放次数统计
  - 最近播放列表

#### 创新功能
- **听歌数据分析**
  - 听歌时长统计
  - 歌手偏好排名
  - 音乐类型分布
  - 听歌时间段分析（早晨、午间、晚上）
  - 情绪曲线分析（基于音乐类型）
  
- **音乐发现**
  - 基于播放历史的推荐
  - 相似歌曲推荐
  - 歌单智能推荐
  
- **音乐日历**
  - 每日推荐歌曲
  - 音乐记忆（某年某月的歌曲回顾）

---

### 1.4 球赛追踪模块 ⚽🏀

#### 基础功能
- **球队关注**
  - 添加关注球队
  - 球队信息管理
  
- **比赛记录**
  - 比赛添加（对手、时间、比分、结果）
  - 比赛类型（联赛、杯赛、友谊赛）
  - 观赛方式记录（现场、直播、回放）
  
- **比赛提醒**
  - 即将开始的比赛提醒
  - 关注球队赛程日历

#### 创新功能
- **观赛数据分析**
  - 观赛场次统计
  - 观赛方式分布（现场vs直播）
  - 支持球队胜率分析
  - 观赛时长统计
  
- **球队数据可视化**
  - 球队赛季战绩图表
  - 进球数/失球数趋势
  - 主客场战绩对比
  
- **观赛日历**
  - 本月观赛日历视图
  - 重要比赛标记

## 二、数据可视化仪表盘 🎯

### 2.1 今日概览
- 今日拍摄照片数
- 今日观看/正在观看的影视
- 今日听歌时长
- 今日/最近的球赛

### 2.2 数据洞察面板
- **时间维度分析**
  - 一周/一月/一年的兴趣活动热力图
  - 各兴趣的时间投入占比饼图
  - 时间趋势折线图
  
- **兴趣关联分析**
  - 观影时听的音乐类型（如果同时记录）
  - 照片拍摄与观影活动的关联
  - 兴趣活动的时间重叠分析

### 2.3 个性化统计
- **年度总结**
  - 今年拍了多少张照片
  - 今年看了多少部电影/剧集
  - 今年听了多少小时音乐
  - 今年看了多少场球赛
  
- **成长曲线**
  - 各兴趣活动的增长趋势
  - 技能提升分析（摄影质量提升）

## 三、数据库设计思路

### 3.1 核心表结构

#### 用户表 (user)
- id, username, password, email, avatar, create_time

#### 摄影相关表
- **photo (照片表)**
  - id, user_id, title, description, file_path, thumbnail_path
  - exif_data (JSON存储), location, tags, category
  - shoot_time, upload_time, view_count, like_count
  
- **photo_album (相册表)**
  - id, user_id, name, description, cover_photo_id, create_time

#### 影视相关表
- **movie (影视表)**
  - id, title, type (电影/剧集), genre, release_year
  - rating, director, actors, description, poster_url
  
- **movie_record (观看记录表)**
  - id, user_id, movie_id, watch_status, personal_rating
  - watch_date, watch_duration, progress, comment, tags
  
- **movie_tag (影视标签表)**
  - id, user_id, tag_name, color

#### 音乐相关表
- **song (歌曲表)**
  - id, title, artist, album, duration, genre, source_url
  
- **playlist (歌单表)**
  - id, user_id, name, description, cover_url, create_time
  
- **playlist_song (歌单歌曲关联表)**
  - playlist_id, song_id, sort_order, add_time
  
- **play_record (播放记录表)**
  - id, user_id, song_id, play_time, duration

#### 球赛相关表
- **team (球队表)**
  - id, name, league, logo_url, country
  
- **user_team (用户关注球队表)**
  - user_id, team_id, follow_time
  
- **match_record (比赛记录表)**
  - id, user_id, home_team, away_team, match_date
  - home_score, away_score, match_type, watch_type
  - venue, notes

#### 标签系统 (通用)
- **tag (标签表)**
  - id, user_id, tag_name, tag_type (photo/movie/music/match)
  - color, create_time

## 四、技术实现方案

### 4.1 后端技术栈
- **Spring Boot 3.x** - 核心框架
- **MyBatis-Plus** - 数据库ORM，简化CRUD
- **MySQL 8.0** - 主数据库
- **Redis** - 缓存（统计数据、热点数据）
- **OSS/MinIO** - 对象存储（照片、音频文件）
- **JWT** - 身份认证
- **WebSocket** - 实时通知（球赛提醒）

### 4.2 前端技术栈
- **Vue 3** - 核心框架
- **Element Plus / Ant Design Vue** - UI组件库
- **ECharts** - 数据可视化
- **Axios** - HTTP请求
- **Vue Router** - 路由管理
- **Pinia** - 状态管理
- **Mapbox / 高德地图API** - 地图展示（摄影地点）
- **PhotoSwipe / Viewer.js** - 照片查看器

### 4.3 核心功能实现

#### 文件上传
- 使用MultipartFile处理文件上传
- 集成OSS服务存储文件
- 图片压缩使用Thumbnailator或ImageIO

#### 数据统计
- MyBatis-Plus的聚合查询
- Redis缓存统计数据，定时更新
- 复杂统计使用SQL视图或定时任务计算

#### 数据可视化
- 后端提供统计API
- 前端ECharts渲染图表
- 支持导出图表为图片

## 五、创新特色功能

### 5.1 智能推荐系统
- **协同过滤算法**
  - 基于用户行为相似性推荐
  - 基于内容相似性推荐
  
- **推荐场景**
  - 基于已看影视推荐新片
  - 基于听歌历史推荐音乐
  - 基于拍摄风格推荐地点/主题

### 5.2 兴趣关联分析
- 分析不同兴趣活动之间的时间关联
- 发现兴趣模式（如：看完球赛后喜欢听什么音乐）
- 个性化兴趣画像生成

### 5.3 时光回顾功能
- **年度回顾**
  - 自动生成年度报告
  - 可视化展示一年来的兴趣活动
  
- **回忆功能**
  - "一年前的今天"你拍了什么照片
  - "这个季节"你常听什么歌
  - 时光轴浏览所有活动

### 5.4 数据导出
- 导出统计数据为Excel
- 导出照片信息为CSV
- 生成年度报告PDF

## 六、用户界面设计

### 6.1 整体布局
- **顶部导航栏** - Logo、导航菜单、用户信息
- **侧边栏** - 四大模块快捷入口

5. **球赛模块** - 关注球队、比赛记录、赛程

- **响应式设计** - 支持PC、平板、手机

## 七、项目亮点总结

1. **多维度数据整合** - 四个兴趣领域的统一管理
2. **深度数据分析** - 不只是记录，更是洞察
3. **可视化展示** - 直观的数据图表和统计
4. **智能推荐** - 基于数据的个性化推荐
5. **兴趣关联** - 发现兴趣之间的关联性
6. **时光记忆** - 记录生活，回顾过去
7. **技术全面** - 涵盖文件上传、数据统计、可视化等
